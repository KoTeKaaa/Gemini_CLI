# Gemini_CLI

`Gemini_CLI` — это продвинутый интерактивный консольный клиент для работы с Gemini через локальный или удалённый FastAPI-сервер. 

Проект поддерживает авторизацию через Supabase, выбор и управление чатами, потоковую выдачу ответов ИИ с красивым рендерингом Markdown, а также **интеграцию с локальной системой (Tool Use / Agentic capabilities)** в безопасном режиме.

---

## ⚡ Ключевые возможности

### 🤖 Интеграция с локальной системой (Tool Use)
ИИ может взаимодействовать с вашим компьютером прямо во время диалога:
- **Чтение файлов (`read_local_files`):** ИИ может запросить содержимое одного или нескольких файлов проекта.
- **Запись и модификация файлов (`write_local_files`):** Позволяет ИИ изменять код или создавать новые файлы с обязательным предпросмотром изменений и подтверждением пользователя.
- **Выполнение команд терминала (`execute_command`):** ИИ может компилировать, запускать тесты или устанавливать библиотеки. Запрос выполняется только после вашего явного одобрения (y/n) и защищён таймаутом в 120 секунд.
- **Песочница (Sandbox):** Проверка путей гарантирует, что ИИ не сможет выйти за рамки текущей рабочей директории без вашего ведома.

### 💻 Удобный CLI-интерфейс (Rich & Prompt-Toolkit)
- **Потоковый Markdown:** Ответы плавно выводятся на экран с полноценной подсветкой синтаксиса и форматированием.
- **Интерактивные диалоги:** Удобное меню выбора чатов и моделей с помощью стрелочек на клавиатуре.
- **ASCII-баннер и статус-бары:** Отображение текущей модели, режима работы и активной рабочей папки.

### 🔒 Безопасность и управление сессией
- **Авторизация:** Вход по email и паролю через Supabase Auth (с поддержкой быстрой регистрации прямо из CLI).
- **Сохранение сессии:** Токены авторизации и конфиг сервера кэшируются локально в папке `~/.gemini_cli/`, избавляя от повторного ввода при перезапуске.
- **Защита выполнения:** Ни одно действие по изменению файлов или запуску команд не произойдет автоматически — клиент всегда запрашивает подтверждение.

### 📂 Управление чатами
- **Постоянные чаты:** История сохраняется в Supabase (таблицы `chats` и `messages`).
- **Временные чаты:** Быстрые сессии без сохранения логов в облаке.
- **Очистка истории:** Поддержка удаления комнат прямо из интерфейса.

## 🛠 Команды внутри CLI

Во время диалога вы можете использовать следующие встроенные команды:

| Команда | Описание |
| :--- | :--- |
| `/help` | Показать интерактивную справку |
| `/model` | Быстро переключить активную модель ИИ |
| `/chat` | Сменить комнату диалога или создать новую |
| `/delete` | Удалить текущий постоянный чат из базы данных |
| `/cd {путь}` | Изменить текущую рабочую директорию проекта для ИИ |
| `exit`, `quit`, `q` | Безопасный выход из приложения |

## 🧱 Структура проекта

- `main.py` — интерактивный CLI-клиент с поддержкой вызова локальных инструментов.
- `server.py` — FastAPI-сервер, выступающий мостом между клиентом, Google Gemini API и Supabase.
- `requirements.txt` — список зависимостей для сборки и работы.

## 🚀 Настройка и запуск

### 1. Подготовка окружения

Рекомендуется использовать виртуальное окружение:

```bash
# Создание окружения
python -m venv .venv

# Активация (Windows)
.\.venv\Scripts\Activate.ps1

# Активация (Linux/macOS)
source .venv/bin/activate

# Установка зависимостей
pip install -r requirements.txt
```

### 2. Настройка сервера

Создайте файл `.env` в корневой папке проекта со следующими параметрами:

```env
API_KEY=your_gemini_api_key
SUPABASE_URL=your_supabase_url
SUPABASE_ANON_KEY=your_supabase_anon_key
```

Запустите сервер (по умолчанию на `http://0.0.0.0:8000`):

```bash
python server.py
```

### 3. Запуск CLI-клиента

В отдельном окне терминала выполните:

```bash
python main.py
```

При первом запуске клиент попросит ввести адрес сервера. Приложение запомнит его и в следующий раз подключится автоматически.

## 📝 Логирование

Все системные события, ошибки подключения и детальные вызовы инструментов логируются в локальный файл:
`~/.gemini_cli/gemini_cli.log`

## 📦 Сборка в исполняемый файл (.exe)

Собрать клиент в один файл можно через PyInstaller:

```bash
pyinstaller --onefile --icon=icon.png -n GeminiCLI main.py
```

Готовый файл появится в директории `dist/`.
