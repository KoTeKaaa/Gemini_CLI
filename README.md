# Gemini_CLI

`Gemini_CLI` — это консольный клиент для работы с Gemini через локальный/удалённый FastAPI-сервер.
Проект поддерживает авторизацию через Supabase, выбор чатов, создание новых комнат диалога и потоковую выдачу ответа модели прямо в терминал.

## Возможности

- Авторизация по email и паролю
- Создание и выбор постоянных чатов
- Временный режим без сохранения истории в БД
- Потоковый вывод ответа модели в консоль
- Переключение модели командой `/model`
- Переключение чата командой `/chat`
- Сохранение сессии пользователя в `~/.gemini_cli`
- Сборка в `.exe` через PyInstaller

## Структура проекта

- `main.py` — CLI-клиент
- `server.py` — FastAPI-сервер, который обрабатывает авторизацию, чаты и запросы к Gemini

## Требования

- Python 3.10+  
- Установленные зависимости:
  - `requests`
  - `prompt_toolkit`
  - `rich`
  - `fastapi`
  - `uvicorn`
  - `google-genai`
  - `python-dotenv`
  - `supabase`
  - `pydantic`

## Настройка сервера

Перед запуском `server.py` нужно создать файл `.env` в корне проекта и указать переменные:

```env
API_KEY=your_gemini_api_key
SUPABASE_URL=your_supabase_url
SUPABASE_ANON_KEY=your_supabase_anon_key
```

## Установка

Рекомендуется запускать проект в виртуальном окружении:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install requests prompt_toolkit rich fastapi uvicorn google-genai python-dotenv supabase pydantic
```

## Запуск

### 1. Запустить сервер на своем VPS/PC

```powershell
python server.py
```

По умолчанию сервер поднимается на `0.0.0.0:8000`.

### 2. Запустить CLI-клиент

В другом окне терминала:

```powershell
python main.py
```

При первом запуске клиент попросит указать адрес сервера, например:

```text
http://{VPS_IP or 127.0.0.1}:8000
```

Если адрес введён без протокола, приложение само подставит `http://`.

## Как пользоваться

После запуска вы увидите меню выбора:

- временный чат
- создание нового чата
- список сохранённых чатов

Внутри чата доступны команды:

- `/model` — сменить модель
- `/chat` — переключиться на другой чат
- `exit`, `quit`, `q`, `выход` — завершить работу

## Поддерживаемые модели

В клиенте доступны следующие базовые модели:

- `gemini-3.1-flash-lite`
- `gemini-3.5-flash`
- `gemini-2.5-flash`
- `gemini-2.5-flash-lite`

## Сборка exe-файла

Сборка выполняется через PyInstaller:

```powershell
pyinstaller --onefile --icon={YOUR_ICON}.png -n {APP_NAME} main.py
```

После сборки исполняемый файл можно взять из папки `dist`.

## Хранение данных

Клиент сохраняет:

- токен сессии — в `~/.gemini_cli/session.json`
- адрес сервера — в `~/.gemini_cli/server_config.json`

## Примечания

- Для постоянных чатов сервер использует Supabase таблицы `chats` и `messages`.
- При ошибке `429` сервер сообщает, что у модели закончились бесплатные токены или лимит запросов, и предлагает сменить модель.
